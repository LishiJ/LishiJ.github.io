Substring && Substr && Slice 的用法和区别

发表于2021-10-14|更新于2021-10-19|note
|阅读量:|评论数:

js 中截取字符串的方式有很多，常见的有 substring substr slice 三种。

对于这三种方法，在用法和功能上很相似，但两者之间的细化区别比较模糊，不太熟悉的小伙伴经常一不小心就用错了，今天我们就分开详解下这三种方法的使用方式和差异。

substring

使用

substring(start, end);

表示返回从 start 开始到 end 处之间的新字符串，其长度为 end 减 start。包含 start，但不包含 end，且不修改原字符串。

注意

  1. 其中 start 是必填项，end 为选填项，如果 end 不填，那表示从 start 截取到字符串结尾。
  1. 如果 start 比 end 大，那么方法在执行前会先交换这两个参数。
  1. 如果 start 与 end 相等，那么会返回一个空的字符串。
  1. 如果 start 或 end 为负数，那么方法在执行前会先将负数变为 0。

substr

使用

substr(start, length)

表示返回从 start 开始截取 length 长度的新字符串，包含 start，且不修改原字符串，与 substring 相比，第二个参数由代表结束的下标 end 变成了规定新字符串长度的 length。

注意

  1. start 是必填项，length 为选填项，如果 length 不填，那表示从 start 截取到字符串结尾。
  1. 如果 start 为负数，那么 start = str.length + start。
  1. 如果 length为 为负数或者0，那么返回空字符串。

slice

使用

slice(start, end)

与 substring 类似，slice 第一个参数为所要截取的字符串的起始下标，end 为所要截取字符串的结束下标，包含 start，不包含 end。

不同的是 slice 不仅可以操作字符串，还可以操作数组，所以 slice 操作不会影响原来的字符串或者数组，返回的也是新的字符串或者数组。

注意

  1. start 是必填项，stop 为选填项，如果 stop 不填，那表示从 start 截取到数组结尾/字符串结尾。
  1. 如果 start 比 stop 大（不会互换），或 start 与 stop 相等（即 start >= end），则截取的为空。
  1. 如果 start 或者 stop 为负数，那么负数的选项从数组尾部开始算起的位置，最后一个数字为-1，倒数第二个数字为-2，与 substr 的 start 相似。

拓展 Splice

使用

splice(start, length, items)

splice 与 substr 有点像，但是 splice 只能操作数组。

表示从下标 start 处截取 length 长度（与）的元素后，在 start 处为原数组添加 items，并返回被截取的新数组，splice 会直接修改原数组。

注意

  1. start 和 length 为必填项，items 为选填项，如果 length 为 0 或者负数，则返回空数组(这里与 substr 相似)。
  1. 如果 start 为负数，则原理和 slice 负数从右往左截取，最后一位数字为-1，倒数第二位为-2，依次类推。
